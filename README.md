# База знаний по LLM, агентам и агентским архитектурам

Репозиторий оформлен как **структурированная база знаний** по большим языковым моделям, агентам, агентским системам и практикам их разработки.

Цель репозитория:
- собирать материалы в одном месте;
- выстраивать понятную навигацию от базовых тем к продвинутым;
- фиксировать как теорию, так и практические паттерны;
- поддерживать единый формат заметок, чтобы знания легко расширялись.

## Как пользоваться репозиторием

1. Начинайте с [обзора](docs/00-overview/index.md), чтобы понять логику структуры.
2. Переходите в нужный тематический раздел и создавайте материалы рядом с `index.md`.
3. Для новых заметок используйте [шаблон статьи](templates/article-template.md).
4. Если добавляете новый важный документ, обновляйте соответствующий `index.md`, чтобы навигация оставалась полной.

## Принципы ведения базы знаний

- **Одна тема — один документ.**
- **Имена файлов должны отражать содержание.**
- **В начале заметки лучше указывать контекст и цель.**
- **Практические рекомендации предпочтительнее абстрактных формулировок.**
- **Каждый раздел должен иметь обзорный `index.md`.**

## Ближайшие направления наполнения

- ключевые понятия и устройство LLM;
- техники prompting и orchestration;
- memory, tools, planning и execution loop у агентов;
- single-agent и multi-agent архитектуры;
- evals, traceability и production-практики;
- сравнение фреймворков и реальных кейсов.
